# Data Layout

- `raw/` stores the original MAT files exactly as provided for the mock lab.
- `interim/` is for reusable intermediate products such as aligned channels, peak selections, frequency axes, and cleaned transmission traces.
- `processed/` is for final exported arrays, CSV files, JSON summaries, or other durable products generated by the Python pipeline.

Recommended rule:

- Keep `raw/` read-only.
- Recompute anything in `interim/` and `processed/` from code.
